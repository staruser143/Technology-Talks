üõ†Ô∏è Freemarker Dispatcher Template

`ftl
<#-- Assume layoutRegistry is loaded into the data model from YAML -->
<#list layoutRegistry as block>
  <#switch block.type>
    <#case "header">
      <h1>${model[block.field]!}</h1>
    <#break>

    <#case "paragraph">
      <p>${model[block.field]!}</p>
    <#break>

    <#case "table">
      <table border="1">
        <tr>
          <#list block.columns as col>
            <th>${col.name}</th>
          </#list>
        </tr>
        <#list model[block.field]! as row>
          <tr>
            <td>${row.item}</td>
            <td>${row.value}</td>
          </tr>
        </#list>
      </table>
    <#break>

    <#case "footer">
      <footer>${model[block.field]!}</footer>
    <#break>

    <#case "chart">
      <div>Render chart of type ${block.chartType} with data: ${model[block.field]!}</div>
    <#break>
  </#switch>
</#list>
`

üîÑ How It Works
- YAML defines the layout schema ‚Üí externalized, editable, auditable.
- Freemarker iterates over schema ‚Üí runtime branching via #switch.
- Model provides data ‚Üí fields mapped dynamically.
- Adding new layouts ‚Üí just extend YAML with a new block type, no template sprawl.
